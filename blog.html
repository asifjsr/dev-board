Question-1: What are the different ways to select an element in the DOM?



In web development, there are several ways to select elements in the DOM: getElementById() selects an element by its id, while getElementsByClassName() and getElementsByTagName() select elements based on their class or tag name, respectively. querySelector() allows you to select the first element matching a CSS selector, and querySelectorAll() selects all matching elements. Additionally, childNodes lets you access all child nodes of an element. These methods provide different options for accessing and interacting with elements on a webpage.

Question-2: What is the difference between innerHTML, innerText, and textContent ?



The main difference between innerHTML, innerText, and textContent lies in how they handle the content inside an element. innerHTML retrieves or sets the HTML markup inside an element, including any HTML tags, while innerText returns the visible text content, ignoring any hidden elements or HTML tags. textContent also retrieves the text inside an element but does not consider any styling or hidden elements, returning all text as-is. While innerHTML is useful for manipulating HTML structures, innerText and textContent are primarily used for working with raw text.

Question-3: What is event delegation in the DOM?



Event delegation is a technique in the DOM where instead of attaching event listeners to individual child elements, you attach a single event listener to a parent element. This listener will then handle events for all current and future child elements that match the event target. It works by taking advantage of event bubbling, where events propagate up the DOM tree. This method improves performance, especially when dealing with a large number of elements, as it reduces the number of event listeners attached to each element.

Question-4: What is event bubbling in the DOM?



Event bubbling in the DOM is when an event triggered on a child element "bubbles up" to its parent elements. This means that the event will first be captured by the innermost element (the one where the event was triggered) and then propagate upwards through its ancestors, triggering any event listeners attached to those parent elements. This allows for more efficient event handling, as you can listen for events on a parent element instead of attaching listeners to each individual child.

Question-5: How do you create, add, and remove elements using JavaScript?



To create an element, use document.createElement('tagName'); add it with parentElement.appendChild(newElement); and remove it with parentElement.removeChild(newElement).
